# SDE-Rank (TIIS Reproducibility Package)

## Quick Start (3 minutes)
### 1) Create environment (Windows + Anaconda)
conda create -n sderank python=3.11 -y
conda activate sderank
pip install -r requirements.txt

### 2) Run (built-in datasets)
python run_experiments.py --dataset email-eu-core --out results
python run_experiments.py --dataset wiki-vote --out results

### 3) Outputs
- Figures: results/figures/
  - FigX_a_gamma.png  (RMSE vs γ, email-eu-core vs wiki-vote)
  - FigX_b_lambda.png (RMSE vs λ, email-eu-core vs wiki-vote)
- Tables: results/tables/
  - TableX_metrics_compare.csv
  - *_metrics.csv, *_sens_gamma.csv, *_sens_lambda.csv

## Project Structure
- run_experiments.py — main entry to run experiments, baselines, ablations, and sensitivity scans
- sde_rank.py — SDE-Rank implementation
- baselines.py — baseline ranking methods (PageRank, HITS, Katz, LeaderRank, etc.)
- ic_simulation.py — Independent Cascade (IC) simulation for influence estimation
- generators.py — synthetic generators for activity/sentiment signals
- data/ — datasets (edge lists)
- results/ — generated outputs (CSV/PNG)

> Note: Some public topologies do not provide message content or temporal activity logs. In such cases, activity and sentiment signals are generated by the provided synthetic generators (see manuscript for assumptions and limitations).
